
С помощью интерфейсов реализовать структуру для обобщения классов модели данных системы, при этом должны быть учтены характеристики:



 - объекты могут быть сущностями или элементами справочника тогда должен быть параметр - внешний код справочника,

 Mutable:
 - изменяемые объекты могут быть изменяемыми извне - тогда должно быть свойство - инициатор последнего изменения,
а могут быть изменямыми - тогда у них должны быть дата последнего изменения и количество изменений,


 Common :
 - объекты могут быть упорядоченными - тогда у них есть номер по порядку,
 - каждый объект является актуальным/устаревшим
 - объекты могут быть неизменными
  - все объекты имеют id и дату создания,
